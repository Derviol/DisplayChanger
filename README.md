# DisplayChanger — 项目说明文档

## 项目概述

DisplayChanger 是一款基于 Python 的 Windows 桌面应用程序，用于自定义修改显示器的分辨率和刷新率。项目采用 tkinter 构建图形界面，通过 ctypes 直接调用 Windows API 实现显示设备的枚举与设置修改，无需依赖任何第三方库，打包后可作为单个 `.exe` 文件分发，开箱即用。

## 技术栈

| 层面     | 技术选型                    |
| -------- | --------------------------- |
| 编程语言 | Python 3.8+                 |
| GUI 框架 | tkinter（标准库）           |
| 系统交互 | ctypes + Windows User32 API |
| 数据建模 | dataclasses                 |
| 打包工具 | PyInstaller 6.0+            |

## 架构设计

整个程序集中在单一源文件 `display_changer.py` 中（约 995 行），按职责自下而上划分为五个层次：

### 1. Windows API 常量与结构体定义

程序通过 ctypes 定义了两个核心 Windows 结构体：

- `DISPLAY_DEVICE`：描述显示设备的基本信息，包括设备名称、描述字符串、状态标志等。
- `DEVMODE`：描述显示模式的完整参数，包括分辨率（dmPelsWidth / dmPelsHeight）、刷新率（dmDisplayFrequency）、色彩深度（dmBitsPerPel）等。

关键 API 常量如 `DM_PELSWIDTH`、`DM_DISPLAYFREQUENCY`、`DISP_CHANGE_SUCCESSFUL` 等均在此层声明。

### 2. 数据模型层

使用 `@dataclass` 装饰器定义 `DisplayInfo` 数据类，用于封装单个显示器的完整信息：

- 设备名称与描述
- 是否为主显示器
- 当前分辨率、刷新率、色彩深度
- 所有支持的显示模式列表
- 支持的分辨率集合
- 各分辨率对应的可用刷新率映射

### 3. 核心业务逻辑层

三个核心函数构成程序的业务引擎：

- **`enumerate_displays()`**：通过 `EnumDisplayDevicesW` 遍历所有活跃且已挂载到桌面的显示设备，再对每个设备调用 `EnumDisplaySettingsW` 枚举全部显示模式，构建完整的 `DisplayInfo` 对象列表。
- **`find_mode()`**：在已枚举的模式列表中查找与目标分辨率和刷新率匹配的 `DEVMODE` 结构体，支持精确匹配和仅分辨率匹配的降级策略。
- **`change_display_settings()`**：调用 `ChangeDisplaySettingsExW` 应用新的显示设置，并对返回码进行中文解析（成功、需重启、不支持的模式、注册表写入失败等）。
- **`revert_display_settings()`**：调用 `ChangeDisplaySettingsExW(None, None, None, 0, None)` 一键恢复原始设置。

### 4. 自定义 Widget 层

为实现现代化的 UI 外观，程序定义了四个自定义 tkinter 组件：

- **`RoundedFrame`**：基于 Canvas 绘制的圆角卡片容器，支持阴影效果。
- **`FlatButton`**：圆角扁平按钮，实现了完整的鼠标悬停（hover）和按下（press）视觉反馈。
- **`SegmentedControl`**：分段选择控件，用于在「常用分辨率」和「自定义输入」两种模式间切换。
- **`StyledEntry`**：带边框、聚焦高亮和占位符（placeholder）提示的输入框组件。

### 5. 主应用层（DisplayChangerApp）

主应用类负责组装全部 UI 并协调交互逻辑，界面采用卡片式布局，包含以下功能区域：

- **标题栏**：蓝色顶栏，显示应用名称与功能描述。
- **显示器选择卡片**：下拉框选择显示器，展示当前分辨率、刷新率、色彩深度和支持模式总数。
- **分辨率设置卡片**：支持两种模式——常用分辨率列表（带搜索过滤，共 36 种标准分辨率，标注支持状态）和自定义宽高输入。
- **刷新率设置卡片**：根据所选分辨率动态列出可用刷新率，支持状态以绿色/红色标签展示。
- **操作按钮区**：「应用设置」、「恢复默认」、「重新检测」三个按钮。
- **状态栏**：带颜色指示点的实时状态文本。

## 核心功能

1. **自动检测显示器**：枚举所有已连接并激活的显示器，识别主显示器。
2. **查看当前参数**：实时展示分辨率、刷新率、色彩深度和支持的模式总数。
3. **常用分辨率快速选择**：内置 36 种从 VGA（640×480）到 8K UHD（7680×4320）的标准分辨率，每项标注当前显示器是否支持。
4. **分辨率搜索过滤**：在常用分辨率列表中支持关键字搜索，快速定位目标分辨率。
5. **自定义分辨率输入**：支持手动输入任意宽高值，自动校验是否为显示器所支持的模式。
6. **智能刷新率列表**：选择分辨率后自动列出该分辨率下所有可用刷新率。
7. **修改前确认**：应用设置前弹出确认对话框，展示当前与目标参数的对比。
8. **一键恢复**：可随时恢复到修改前的原始设置。
9. **15 秒安全回滚**：Windows 系统在设置后 15 秒内无确认将自动恢复，防止黑屏。
10. **打包分发**：通过 `build.bat` 一键打包为单个 `.exe` 文件，目标机器无需安装 Python。

## 设计主题

程序定义了完整的 `Theme` 类管理配色方案，整体采用浅色背景 + 蓝色主色调的设计语言：

- 背景色：`#f5f6fa`（浅灰）
- 卡片背景：`#ffffff`（白色）
- 主色调：`#3498db`（蓝色）
- 危险色：`#e74c3c`（红色，用于恢复按钮和不支持标签）
- 成功色：`#27ae60`（绿色，用于状态指示和支持标签）

## 系统要求与兼容性

- 操作系统：Windows 7 / 8 / 10 / 11
- 运行 `DisplayChanger.exe` 无需任何额外依赖
- 从源码运行仅需 Python 3.8+，无需安装第三方包（tkinter 和 ctypes 均为标准库）
- 建议以管理员权限运行以确保设置修改权限

## 注意事项

- 修改分辨率和刷新率前，请确认显示器支持目标参数，设置过高刷新率或不支持的分辨率可能导致黑屏。
- 如出现黑屏，Windows 系统会在 15 秒内自动恢复原始设置。
- 自定义输入的分辨率必须是显示器已通过 `EnumDisplaySettingsW` 枚举到的模式之一。
